# Living README Intelligence Canon

A serious AndyAI repo is not a folder with files. It is a knowledge front page.

A living README must show what the system is, what it knows, how knowledge is organized, what is canonical, what is draft, what is archived, what is export-ready, what the operator should do next, where proof and evidence live, and how beginners should enter the domain.

Canonical rule: README is the first visible intelligence surface of the repository.
